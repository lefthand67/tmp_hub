# Список литературы для AI-инженерии

🏆 Уровень «Обязательно к прочтению» (Наивысший приоритет)
Эти книги максимально соответствуют вашим техническим требованиям и архитектурным задачам.

1. **AI Agents: The Definitive Guide - Design, Deployment, and Evaluation** — Nicole Koenigstein
   - **Почему это важно:** Эта книга напрямую отвечает вашим основным потребностям. В ней подробно рассматривается создание строгих конвейеров оценки агентов, управление циклами управления агентом (такими как ReAct и Reflexion) и обеспечение безопасности исполнения.
   - **Значимость для проекта:** Оценка агента, перемещающегося по графу пакетов, невероятно сложна, так как сбой может произойти на этапе поиска, рассуждения или вызова инструментов. В этой книге прямо рассматривается проблема «имитации агентности» (agent-washing) и создание надежных фреймворков оценки для агентов промышленного уровня.

2. **AI Model Evaluation (MEAP)** — Leemay Nassery
   - **Почему это важно:** Эта книга восполняет серьезный пробел, выходя за рамки академических метрик точности и анализируя работу систем в реальном мире.
   - **Значимость для проекта:** Она охватывает офлайн-диагностическую оценку, использование LLM в качестве автоматизированных судей (LLM-as-a-judge), теневой трафик и тестирование системных задержек. Поскольку парсинг глобального графа пакетов Debian может легко стать узким местом с точки зрения задержек, критически важно понимать, как оценивать как семантическую точность модели, так и инженерную производительность системы.

3. **Retrieval Augmented Generation, The Seminal Papers (MEAP)** — Ben Auffarth
   - **Почему это важно:** Вы строите систему GraphRAG. Для отображения сложных сущностей, таких как зависимости пакетов Debian (Depends, Conflicts, Provides, Breaks), простых векторных эмбеддингов будет недостаточно.
   - **Значимость для проекта:** Понимание теоретических и фундаментальных механизмов архитектуры RAG «с нуля» сэкономит вам недели архитектурного перепроектирования. Это поможет создать надежные и проверяемые механизмы подачи контекста еще до того, как ваш агент приступит к работе с данными.

🔬 Уровень «Исследования и метрики» (Дополнительные материалы)
Внутренние исследования Anthropic по вопросам оценки агентов.

- **Demystifying evals for AI agents** — Anthropic (Jan 09, 2026)
  - Ссылка на файл: `evals/demystifying_evals_for_ai_agents/demystifying_evals_for_ai_agents___anthropic.md`
  - **Описание:** Глубокий анализ проектирования оценок (evals) для агентов, включая использование различных типов грейдеров (code-based, model-based, human) и различие между capability и regression evals.

- **Quantifying infrastructure noise in agentic coding evals** — Anthropic (Feb 05, 2026)
  - Ссылка на файл: `evals/quantifying_infrastructure_noise_in_agentic_coding_evals/quantifying_infrastructure_noise_in_agentic_coding_evals___anthropic.md`
  - **Описание:** Исследование влияния инфраструктурных настроек на результаты бенчмарков для агентов (таких как Terminal-Bench и SWE-bench). Статья доказывает, что конфигурация ресурсов может влиять на оценку моделей сильнее, чем реальная разница в их способностях.

🏗️ Уровень «Структурный контекст» (Читать выбранные главы)
Эти книги очень ценны, но их следует использовать как справочный материал или читать выборочно для архитектурного проектирования.

4. **Learning Systems Thinking** — Diana Montalion
   - **Чем полезна:** Это не книга по ИИ, но это важнейший учебник по архитектуре программного обеспечения для вашей конкретной области.
   - **Значимость для проекта:** Управление релизами дистрибутивов Linux и деревья зависимостей пакетов являются классическими нелинейными сложными системами. Когда изменение одного пакета вызывает каскад изменений в 50 зависимых пакетах, стандартное линейное мышление перестает работать. Используйте эту книгу, чтобы спроектировать то, как ваша система рассуждает об изменениях состояния между релизами Debian.

5. **AI Engineering: Building Applications with Foundation Models** — Chip Huyen
   - **Чем полезна:** Чип Хьюен — мастер паттернов проектирования машинного обучения промышленного уровня. Хотя некоторые части книг по базовым моделям быстро устаревают, ее архитектурные принципы построения конвейеров данных, стратегии кэширования и оркестрации систем остаются базовыми для ИИ-инженерии. Прочитайте ее, чтобы создать фундаментальный чертеж проектирования вашей системы.
